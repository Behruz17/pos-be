# POS Backend System

Backend-приложение для системы управления точкой продаж (Point of Sale), реализованное на Node.js с использованием Express.js и MySQL.

## 📋 Описание

Это полнофункциональная система управления продажами, складскими остатками и клиентами. Приложение предоставляет RESTful API для интеграции с фронтенд-приложениями.

## 🚀 Основные возможности

### 🔐 Аутентификация и авторизация
- Вход/выход пользователей
- Токен-базированная аутентификация (JWT-style)
- Ролевая система: ADMIN и USER
- Многоуровневый контроль доступа:
  - **Администраторы**: Полный доступ ко всему
  - **Пользователи**: Полный доступ только к своему магазину, read-only доступ к остальным
- Защита всех эндпоинтов через middleware
- Привязка пользователей к магазинам (store_id)

### 👥 Управление пользователями
- Создание, чтение, обновление, удаление пользователей (только для администраторов)
- Регистрация новых пользователей
- Обязательная привязка USER-пользователей к магазину
- Просмотр списка всех пользователей с указанием store_id

### 🏢 Управление складами
- Создание и удаление складов
- Просмотр товаров на конкретных складах
- Поддержка нескольких складов
- Автоматическое создание основного склада при необходимости

### 📦 Управление товарами
- Создание и удаление товаров
- Просмотр каталога товаров
- Отображение последних цен и общих остатков
- Учет производителя и изображений товаров
- Загрузка изображений как в виде URL, так и файлов с локального компьютера при создании или обновлении товаров

### 📥 Инвентаризация (приход товаров)
- Создание документов прихода
- Детализация по товарам с учетом коробок и штучных единиц
- Учет веса и объема
- Автоматическое обновление остатков на складе
- История всех приходов

### 📊 Управление остатками
- Просмотр остатков на всех складах
- Корректировка остатков с указанием причины
- Перемещение товаров между складами
- Полная история изменений остатков

### 👤 Управление клиентами
- Создание и управление клиентами
- Отслеживание баланса клиентов
- Просмотр истории транзакций (продажи и возвраты)
- Ручная корректировка баланса

### 💰 Продажи
- Создание продаж с автоматическим списанием товаров
- Поддержка продаж без указания клиента (используется демо-клиент)
- Распределение списания между всеми записями остатков товара
- Автоматическое обновление баланса клиентов

### 🔄 Возвраты
- Создание возвратов товаров
- Связь с исходными продажами
- Автоматическое возвращение товаров на склад
- Обновление баланса клиентов

## 🛠 Технологии

- **Node.js** - среда выполнения JavaScript
- **Express.js** - веб-фреймворк
- **MySQL** - реляционная база данных
- **bcrypt** - хэширование паролей
- **cors** - поддержка CORS
- **dotenv** - управление переменными окружения

## 🚀 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <repository-url>
cd pos-be
```

### 2. Установка зависимостей
```bash
npm install
```

### 3. Настройка базы данных
Создайте файл `.env` в корне проекта:
```env
DB_HOST=localhost
DB_USER=your_username
DB_PASSWORD=your_password
DB_NAME=pos_database
PORT=3000
```

### 4.1 Миграция базы данных (для включения контроля доступа)
Выполните миграцию для добавления поддержки привязки пользователей к магазинам:
```sql
-- Выполните содержимое файла migration_add_store_id_to_users.sql
```

После миграции назначьте каждому USER-пользователю соответствующий store_id.

### 4. Создание структуры базы данных
Импортируйте структуру из файла `db.sql` в вашу MySQL базу данных.

### 5. Запуск приложения
```bash
# Для production
npm start

# Для разработки (с авто-перезагрузкой)
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`

## 📖 Документация API

Полная документация всех эндпоинтов находится в файле [api_documentation.md](api_documentation.md).

Документация системы контроля доступа: [ACCESS_CONTROL.md](ACCESS_CONTROL.md)

Основные категории API:
- `/api/auth/*` - Аутентификация
- `/api/users/*` - Управление пользователями
- `/api/stores/*` - Управление магазинами
- `/api/warehouses/*` - Управление складами
- `/api/products/*` - Управление товарами
- `/api/inventory/*` - Инвентаризация
- `/api/warehouse/stock/*` - Управление остатками
- `/api/stock/history/*` - История изменений
- `/api/customers/*` - Управление клиентами
- `/api/sales/*` - Продажи
- `/api/returns/*` - Возвраты
- `/api/delivery-drivers/*` - Управление доставщиками

## 🔧 Разработка

### Структура кода
- **server.js** - содержит все маршруты API
- **db.js** - настройка пула соединений с MySQL
- **middleware.js** - проверка токенов аутентификации и контроль доступа

### Контроль доступа
- **authMiddleware** - базовая аутентификация
- **authorizeStoreAccess** - контроль доступа на уровне магазинов
- Все USER-пользователи должны иметь store_id в таблице users
- Администраторы могут работать со всеми магазинами
- Пользователи автоматически ограничены своим магазином

### Особенности реализации
- Использование транзакций MySQL для обеспечения целостности данных
- Централизованная обработка ошибок
- Валидация входных данных
- Логирование всех операций

## ⚠️ Важные замечания

1. Все API эндпоинты требуют авторизации через заголовок `Authorization: Bearer <token>`
2. USER-пользователи должны быть привязаны к магазину (store_id)
3. Удаление товаров и складов возможно только при отсутствии связанных записей
4. При продажах система автоматически распределяет списание между всеми доступными остатками товара
5. При возвратах товары всегда возвращаются на основной склад
6. Для включения системы контроля доступа необходимо выполнить миграцию базы данных
